Agent 核心由目标理解与决策、工具调用能力、循环执行框架组成。没有闭环执行，就无法形成真正的 Agent。

企业级 Agent 通常需要接入层、路由层、Agent 核心层、知识层、数据层和基础设施层。接入层负责鉴权、限流、请求入口治理；路由层负责会话管理和 Agent 分发。

Memory 负责维护多轮对话上下文，Session 负责隔离不同用户和不同会话。Memory 让 Agent 记住内容，Session 让系统知道这些记忆属于谁。

RAG 用于补足模型参数外的企业知识，让 Agent 在知识边界内回答更稳定。一个最小可用的 RAG 系统可以从文档切片、检索、重排、上下文拼接开始。

Tools 的价值不只在数量，更在可靠性、幂等性、错误处理、超时控制和结果结构化。高质量工具会显著提升 Agent 的稳定性。

企业级运行时需要最大循环次数、工具异常隔离、请求标识、限流和可观测性。稳定性组件不改变 Agent 本质，但决定系统是否可在生产环境中持续运行。
